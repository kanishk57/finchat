FinChat is a financial document analysis workspace built around retrieval-augmented generation. It ingests PDFs, indexes them with a hybrid search stack, reranks retrieved chunks, and streams cited answers back to the user. It supports both a web interface and an interactive console mode.

## 🛠️ Pipeline Technical Deep Dive

### 1. Ingestion & Chunking
* **Parsing**: Raw PDFs are processed with `pypdf.PdfReader` at the page level. Modification times on disk are converted to document metadata timestamps.
* **Cleaning**: Repeats of whitespace, blank lines, and formatting anomalies are normalized.
* **Chunking Strategy**: Text is split by character lengths with sliding window overlap. The splitter searches for boundaries (periods, newlines) near the edge of thresholds to preserve semantic integrity, ignoring tiny fragments.
* **Metadata Schema**:
  ```json
  {
    "doc_name": "filename.pdf",
    "doc_date": "YYYY-MM-DD",
    "page_number": 4,
    "chunk_id": "filename_page4_chunk1",
    "text": "... enriched text containing page & doc headers ..."
  }
  ```

### 2. Hybrid Retrieval & Indexing
* **Dense Retrieval (FAISS)**: Uses `BAAI/bge-base-en-v1.5` under a `passage:` instruction prefix. Vector mappings are normalized using $L2$ norms and loaded into a flat inner-product index (`IndexFlatIP`).
* **Sparse Retrieval (BM25)**: Uses the BM25 model based on tokenized corpus documents.
* **Hybrid Combination**: Dense semantic search retrieves top candidates, while lexical search collects exact query terms. Results are deduplicated by `chunk_id` and combined. Duplicate hits receive a scoring boost.

### 3. Cross-Encoder Reranking
* Candidates are processed in query-passage pairs by the `BAAI/bge-reranker-base` cross-encoder.
* Raw output scores are normalized using a sigmoid function:
  $$\text{Score} = \sigma(x) = \frac{1}{1 + e^{-x}}$$
* Chunks scoring below the threshold are filtered out, and the remaining top results are passed to context preparation.

### 4. LLM Generation Layer
* **Prompt Assembly**: Formats context into a clean instruction template emphasizing precise reference tracking and deduplicated page citations.
* **Model Communication**: Relies on the `/v1/chat/completions` API from a local `llama.cpp` server. Fallback paths exist to `/completion` for compatibility with older interfaces.
* **SSE Streaming**: Sends citation metadata immediately before streaming the generated answer chunks to the frontend.

## 📡 API Surface (FastAPI)

All endpoints are hosted under `/api/v1`:

### Chat Routes (`/api/v1/chat`)
* **`POST /`**: Evaluates queries. Detects `@filename` syntax to restrict searches, manages session-specific questions, retrieves context, and streams SSE citations/tokens.
* **`DELETE /session/{session_id}`**: Purges user files and vectors from memory, updates indexes, and removes directories.
* **`POST /export`**: Re-formats session HTML logs into Markdown summaries, rewrites tags, writes to `/tmp`, and triggers browser downloads.

### Document Routes (`/api/v1/documents`)
* **`POST /upload`**: Accepts PDF payloads. Re-indexes local directory vectors or handles isolated sub-indexing.
* **`GET /`**: Lists available files (global and session-scoped).
* **`DELETE /{filename}`**: Deletes the specified file and initiates background index updates.
* **`GET /progress`**: Exposes background indexing status values to the UI.
* **`GET /portfolio/stats`**: Aggregates upload sizes, count metrics, entity details, and timestamps.

## 💻 Runtime Modes

### 1. Web Application Mode (Primary)
The system boots a FastAPI app, registers routes, sets up CORS controls, mounts file-viewing endpoints, and loads the RAG system into the memory pool on startup.
```bash
chmod +x run.sh
./run.sh
```

### 2. Interactive Console Mode
Run the RAG loop entirely within the terminal. Perfect for low-overhead local testing.
```bash
python main.py
```
